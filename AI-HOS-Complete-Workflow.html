3. Authentication Workflows

Hospital Staff Login (PIN-based)

User selects role & branch → Enters PIN → Server validates PIN against tenant record → Rate limit: 5 attempts / 15 min (Redis) → JWT issued (12h expiry) → Custom Supabase JWT for RLS → Redirect to role dashboard

Hospital Staff Login (Password-based)

User enters email + password → Supabase Auth validates credentials → Lookup user_credentials table for role mapping → Verify tenant is active → JWT issued with role/tenant

Patient Login (OTP via WhatsApp)

Patient enters phone number → API checks patient exists (phone variants: 10-digit + 91-prefixed) → Rate limit: 3 OTPs / phone / 15 min + 10 OTPs / IP / 15 min → 6-digit OTP generated & stored in patient_otps table → OTP sent via WhatsApp Cloud API → Patient enters OTP (5 min validity) → Server verifies OTP (both phone formats matched) → JWT issued with PATIENT role → Redirect to /patient dashboard

4. Reception & Queue Management

Walk-in Patient Booking

Reception opens /reception/book → Search existing patient or create new → Select doctor & time slot → Booking created with token number → Patient added to queue (status: WAITING) → WhatsApp confirmation sent to patient → Token receipt printable

Queue Board (/reception)

  • Real-time stats banner: Total waiting, in-consultation, completed, avg wait time
  • Queue cards: Token #, patient name, doctor, wait time (elapsed timer)
  • Status flow: WAITING → CHECKED_IN → IN_CONSULTATION → COMPLETED
  • Quick actions: Check-in, Mark arrived, Cancel, Reschedule
  • Real-time updates: Supabase realtime subscriptions

Appointment Management (/reception/appointments)

  • Calendar view of all appointments
  • Filter by doctor, date range, status
  • Reschedule and cancel with WhatsApp notification
  • Pending arrivals panel with quick check-in

Patient Search (/reception/patients)

  • Search by name, phone, patient ID
  • View patient history, past visits, documents
  • Quick book appointment from patient record

5. Doctor Consultation Workflow

Doctor Dashboard (/doctor)

  • Today's queue: patients waiting, in consultation
  • Stats: total patients today, completed, pending
  • Quick access to current patient

Consultation Flow (/doctor/consult)

Doctor opens consult page → Sees queue of checked-in patients → Clicks patient to start consultation → Records: Chief complaints, diagnosis, vitals, clinical notes → Writes prescription (medicine combobox with templates) → Orders lab tests (if needed) → Admits to IPD (if needed) → Video call button (telemedicine, feature-gated) → Marks consultation complete → Post-consultation notification sent via WhatsApp → Prescription printable / PDF exportable

EMR - Electronic Medical Record (/doctor/emr/[phone])

  • Complete patient history by phone number
  • Past consultations, prescriptions, lab results, vitals timeline
  • Document gallery (uploaded reports, images)

Prescription Management

  • Medicine combobox with auto-complete
  • Prescription templates (save & reuse common prescriptions)
  • Print prescription (branded hospital letterhead)
  • Download as PDF

Doctor Schedule (/doctor/schedule)

  • Set weekly availability (day + time slots)
  • Block specific dates
  • Visible to patients during booking

6. Pharmacy Module

Feature-gated: Medium & Enterprise tiers

Prescription Orders (/pharmacy)

Doctor writes prescription → Order appears in Pharmacy Kanban → Pharmacist sees: Pending → Preparing → Ready → Dispensed → Pharmacist moves cards through columns → Patient notified when ready for pickup

Inventory Management (/pharmacy/inventory)

  • Medicine stock tracking
  • Low stock alerts
  • Batch and expiry tracking
  • Add/edit/delete medicines

7. Lab Module

Feature-gated: Medium & Enterprise tiers

Lab Order Workflow (/lab)

Doctor orders lab test during consultation → Order appears in Lab Kanban board → Lab Tech sees: Pending → Sample Collected → In Progress → Completed → Lab Tech enters results with normal range indicators → Results visible to Doctor in EMR → Results visible to Patient in portal → Lab report printable with normal ranges

8. Inpatient (IPD) Module

Feature-gated: Enterprise tier only

Admission Flow

Doctor/Reception initiates admission from consult → Admit dialog: select bed, ward, admission notes → Patient status changes to ADMITTED → Appears on Inpatient board (/reception/admissions)

9. Billing & Payments

Billing Workflow (/admin/billing)

Consultation completed → Invoice auto-generated (consultation fee + lab + pharmacy + procedures) → GST/tax calculation (lib/billing/tax.ts) → Payment: Cash / Razorpay (online) → Razorpay callback verifies payment → Invoice printable / PDF → Payment receipt sent via email

Invoice Print Components

  • Invoice with hospital branding, GST, itemized charges
  • OP consultation summary print
  • Patient card print
  • All printable via PrintButton or DownloadPdfButton

12. WhatsApp AI Agent

Feature-gated: Medium & Enterprise tiers

Architecture

Patient sends WhatsApp message → Meta Webhook → /api/whatsapp/webhook → Message deduplication → Session management (conversation context) → Claude AI Agent processes message → Tool calls: book appointment, check status, get prescriptions, etc. → Response sent back via WhatsApp Cloud API

AI Agent Tools

ToolWhat it does
book_appointmentBook appointment with doctor
check_appointment_statusCheck upcoming appointments
cancel_appointmentCancel an existing appointment
get_prescriptionsRetrieve recent prescriptions
get_lab_resultsRetrieve lab test results
get_doctorsList available doctors & specialties
get_available_slotsCheck doctor availability
get_queue_statusCheck current queue position

Multi-language Support

The WhatsApp agent responds in the language the patient uses (English, Hindi, Telugu). Translation layer in lib/whatsapp/translations.ts.

14. Telemedicine

Feature-gated: Medium & Enterprise tiers

Video Consultation Flow

Doctor clicks "Video Call" during consultation → Jitsi Meet room created (unique per appointment) → Full-screen video dialog opens in browser → "Copy Link for Patient" copies Jitsi URL → Link sent to patient via WhatsApp API → Patient joins from their phone/computer → Doctor can consult, share screen → No API keys needed (Jitsi is free/open source)

  • Service Worker with stale-while-revalidate caching
  • Installable on desktop, Android, iOS
  • PNG icons (192px, 512px, Apple touch icon)
  • Push notification support
  • App shortcuts: Queue Board, Book Appointment

i18n (Internationalization)

  • 3 languages: English, Hindi, Telugu
  • 95 translation keys across all modules
  • Language switcher in sidebar
  • Persisted in localStorage

Security

  • Redis rate limiting on all auth endpoints
  • Supabase RLS for tenant data isolation
  • Custom JWT for client-side Supabase access
  • Security headers: HSTS, X-Frame-Options: DENY, CSP
  • OTP anti-enumeration (generic error messages)
  • PIN brute-force protection (5 attempts / 15 min)
